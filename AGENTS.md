
# 设计文档：Instant-Spatial-3D (IS3D) 跨平台生成器

## 1. 项目概述
### 1.1 目标
从单张 RGB 图像出发，在 1 秒内生成一个具有真实物理尺度（Metric Scale）、支持自由漫游、且具备遮挡区域自动补全（Inpainting/Hallucination）功能的 3D 高斯场景（3DGS）。

### 1.2 核心特性
*   **瞬时性**：采用 Feedforward（前向传播）架构，无需每张图优化。
*   **度量准确**：生成的 3D 空间坐标以“米”为单位，适配 AR/VR。
*   **生成式补全**：利用生成式先验补全照片背后的视角（Out-painting）。
*   **跨平台渲染**：支持 Web、iOS、Android 和 PC 端的实时查看。

---

## 2. 技术架构 (System Architecture)

系统分为三个核心层：**感知与编码层**、**高斯回归与补全层**、**跨平台渲染层**。

### 2.1 算法流程图
`输入图片` -> `特征提取 (ViT)` -> `深度与尺度估计` -> `高斯参数预测` -> `背面/遮挡补全` -> `3DGS 导出 (.ply)` -> `跨平台渲染器`

---

## 3. 核心模块设计

### 3.1 感知与编码模块 (Perception & Encoding)
*   **Backbone**: 采用 **DINOv2** 或 **ViT-L/14**。这些模型在预训练阶段已经掌握了极强的空间几何感。
*   **内参估计器 (Global Header)**：预测相机的焦距 (f) 和主点位置，这是将 2D 转换为 3D 度量空间的基石。

### 3.2 高斯回归模块 (3DGS Regression) —— 核心 SHARP 策略
*   **像素级预测**：对于输入图的每个像素，网络预测：
    *   **$\mu$ (位置)**: 基于深度图反投影生成的 3D 坐标 $(x, y, z)$。
    *   **$\Sigma$ (各向异性斜方差)**: 缩放 ($s$) 和旋转 ($q$)，决定高斯球的形状。
    *   **$\alpha$ (不透明度)**: 区分物体边缘。
    *   **$C$ (球谐函数/颜色)**: 捕捉光影变化。
*   **绝对尺度实现**：在损失函数中引入 **Metric Depth Loss**，利用带真实尺寸的数据集（如 Matterport3D）强制模型学习物理单位。

### 3.3 补全与幻觉模块 (Hallucination & Completion)
单张照片只能看到“正面”，要实现漫游，必须补全“背面”：
*   **对称性与生成先验**：引入类似 **Zero-1-to-3** 或 **Stable Diffusion 3D** 的生成逻辑。
*   **预测层级**：不仅为可见像素生成高斯球，还要预测一个“潜在空间”中的高斯分布，专门负责遮挡区域和图像边缘外的补全。
*   **策略**：训练一个 Transformer Decoder，查询（Query）未观测区域的空间坐标，输出该位置可能存在的高斯属性。

## 4. 跨平台实现方案 (Cross-Platform Strategy)

为了让生成的 3D 空间在不同设备上流畅运行，需要采用“云端/本地推理 + 通用渲染”的策略。

### 4.1 模型部署
*   **后端/云端 (Python/PyTorch)**：负责高性能推理。
*   **移动端推理**：
    *   **iOS**: 通过 **CoreML** 转换模型，利用 Apple Neural Engine (ANE) 加速。
    *   **Android**: 使用 **TensorRT** 或 **ONNX Runtime**。
*   **Web**: 通过 **WASM + WebGPU** 运行轻量化模型。

### 4.2 实时渲染器 (Client Renderer)
由于 SHARP 输出的是标准 3DGS (.ply)，渲染端可以选择：
*   **Web**: 使用 `gsplat.js` (基于 WebGPU/WebGL)。
*   **移动端 (Native)**：使用 C++/Metal (iOS) 或 C++/Vulkan (Android) 编写的高斯喷射渲染器。
*   **Unity/Unreal**: 使用现成的 3DGS 插件进行漫游环境搭建。

## 5. 数据集与训练策略

### 5.1 数据来源
1.  **RE10K / RealEstate10K**: 学习室内漫游的几何关系。
2.  **ScanNet / Matterport3D**: 提供带真实物理单位（米）的深度图，用于尺度对齐。
3.  **Objaverse**: 学习物体的背面补全先验。

### 5.2 损失函数 (Loss Functions)
*   **$\mathcal{L}_{render}$**: 渲染出的新视角与真值的像素差异。
*   **$\mathcal{L}_{metric\_depth}$**: 深度值的物理准确性损失。
*   **$\mathcal{L}_{gaussian\_reg}$**: 正则化项，防止生成过大或重叠过多的高斯球。

---

## 6. 漫游交互设计 (Interaction Design)

1.  **第一人称视角 (FPV)**：初始位置设为相机原点 $(0,0,0)$。
2.  **碰撞检测**：利用生成的高斯点云生成一个简化的导航网格（NavMesh），防止漫游时穿模。
3.  **动态加载**：对于大场景，按距离裁剪高斯球的数量，保证移动端帧率稳定在 60FPS。

---

## 7. 实施路线图 (Roadmap)

*   **Phase 1 (原型期)**：基于现有的 SHARP 开源代码，实现单图到 .ply 的稳定转换。
*   **Phase 2 (尺度优化)**：引入度量深度估计，确保生成物在 AR 下大小正确。
*   **Phase 3 (生成补全)**：加入生成式 Decoder，实现 360 度视角补全（即使背面是模糊的）。
*   **Phase 4 (全平台部署)**：开发 Web 端查看器和 iOS 原生演示 App。

---

## 8. 关键挑战与解决方案

| 挑战 | 解决方案 |
| :--- | :--- |
| **遮挡区“融化”感** | 引入对抗性损失 (GAN Loss) 或 Diffusion Refinement 提升纹理细节。 |
| **跨平台算力差异** | 提供多级模型：Lite 版（手机端，10万高斯点）与 Pro 版（云端，100万高斯点）。 |
| **光照不一致** | 预测环境光图（Environment Map），对补全区域进行光照调整。 |
